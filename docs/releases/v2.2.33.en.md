# Twitch Drop Farmer v2.2.33

This release adds log timestamps and fixes the dashboard only ever showing the actively-farmed game.

## Highlights

- Every log line now shows a `[HH:MM:SS]` prefix, making it possible to tell real log spam apart from legitimately repeated messages spread out over time.
- `fetch_campaigns` used to return only what a single call discovered. Most poll cycles run off the main thread and can't use the browser fallback, so they only see the 1-2 in-progress campaigns from Inventory — and that thin result was replacing whatever richer list a previous cycle had found, so only the actively farmed game ever appeared populated.

## Added

- `TwitchClient._campaign_cache`: a lock-protected dict that every `fetch_campaigns()` call merges into (instead of replacing), pruning campaigns that are now clearly expired. Returned campaigns are copies (`dataclasses.replace`) so concurrent callers on different threads can't bleed in-place mutations into each other or into the cache.

## Result

Version 2.2.33 lets all whitelisted games fill in with real data over time, even slowly, instead of only the one currently being farmed — plus timestamps to make future log analysis easier.
