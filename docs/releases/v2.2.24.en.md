# Twitch Drop Farmer v2.2.24

This release tightens streamless progress reconciliation and closes a few release hygiene gaps, with focus on real progress and packaging consistency.

## Highlights

- More robust streamless progress reconciliation when Twitch delays inventory updates.
- Extra `CurrentDrop` signal to reduce false "no progress" states.
- Improved internal drop mapping for more reliable diagnostics.
- Cleanup of locally generated release artifacts.

## Fixes

- Prevented apparent progress stalls when `sendSpadeEvents` succeeds but Inventory has not yet reflected the latest watched minutes.
- Fixed embedded JS regex escapes to remove runtime warnings during startup/tests.

## Result

Version 2.2.24 is aligned with the current codebase and the changelog entries for this release.
