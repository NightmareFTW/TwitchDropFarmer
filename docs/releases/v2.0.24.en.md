# Twitch Drop Farmer v2.0.24

This release closes a major stability and UX cycle, focused on campaign resilience, dashboard consistency, and better diagnostics flows.

## Highlights

- Manual game targeting is now sticky and more predictable.
- "Hide subscription-only drops" filtering is more reliable, including metadata-sparse campaigns.
- Dashboard grid compacts correctly after hidden-game filtering (no blank gaps).
- Diagnostics now include a compact per-test report (status, duration, message).
- Diagnostic/update buttons provide better feedback and safer operation handling.

## Fixes

- Fixed filter tab title/count refresh regression.
- Fixed false OAuth critical diagnostics for durable session (JSON) users.
- Prevented window shutdown during diagnostics by removing rendered browser fallback (Qt WebEngine) from worker-thread diagnostic path.

## Security and privacy

- Pre-release review performed to detect tracked sensitive artifacts.
- Credentials and sessions remain local in `~/.twitch-drop-farmer/`.
- `cookies.json`, `campaign_cache.json`, and user session files remain outside repository tracking.

## Result

Version 2.0.24 is release-ready with stronger operational stability, cleaner diagnostics, and improved dashboard UX.
