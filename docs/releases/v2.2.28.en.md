# Twitch Drop Farmer v2.2.28

This release fixes a dashboard-only regression that mislabeled almost every campaign as requiring a subscription.

## Highlights

- Fixed global false-positive "subscription required" warning on the dashboard, active-drops list, and hide-sub-only checkbox count.
- Root cause was UI-layer only; the actual farming decision engine was never affected.

## Fixes

- `_campaign_matches_hide_sub_only` no longer treats campaigns with missing per-drop metadata (common for browser-fallback campaigns, when Twitch's integrity check blocks `DropCampaignDetails`/`ViewerDropsDashboard`) as subscription-locked.
- It now delegates solely to `_campaign_is_known_subscription_locked`, which only fires on explicit subscription signals.

## Result

Version 2.2.28 restores correct subscription-only labeling so real, farmable campaigns are no longer hidden or flagged by mistake.
