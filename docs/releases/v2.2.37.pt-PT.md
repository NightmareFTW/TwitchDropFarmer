# Twitch Drop Farmer v2.2.37

Esta versao transforma o botao de atualizar o dashboard num scan explicito e completo de detalhe de toda a whitelist.

## Destaques

- O botao "Atualizar dashboard" antes so limpava caches de imagens e corria um poll normal, continuando limitado ao orcamento habitual de 8 campanhas com detalhe por ciclo. Seleccionar outro jogo logo a seguir a actualizar continuava a mostrar dados de drops vazios na maioria das vezes.

## Adicionado

- `fetch_campaigns(full_scan=True)` / `FarmEngine.poll(full_scan=True)`: um pedido explicito que levanta o limite de deteccao por ciclo, para que todas as campanhas da whitelist sem dados reais por-drop recebam uma visita a pagina de detalhe de uma so vez, nao so um punhado. O botao de atualizar o dashboard passa agora a chamar isto de forma sincrona na thread principal (obrigatorio para o fallback do browser), desactivando-se a si proprio e registando o inicio/fim no log, ja que pode demorar alguns minutos com uma whitelist grande. Verificado com um teste isolado: 19/20 campanhas recebem uma visita de detalhe num scan completo, contra 8/20 num ciclo normal.

## Resultado

A versao 2.2.37 permite ao utilizador "aquecer" proactivamente todo o dashboard antes de mudar entre jogos, em vez de so o que esta a ser farmado activamente mostrar dados reais.
