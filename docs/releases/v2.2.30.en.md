# Twitch Drop Farmer v2.2.30

This release fixes a critical bug that made nearly every request fail Twitch's integrity check and forced the app into a slow browser-based fallback.

## Highlights

- Fixed `WEB_CLIENT_ID` and `ANDROID_APP_CLIENT_ID` being empty strings, which caused every direct GraphQL call (Inventory, ViewerDropsDashboard, DropCampaignDetails, etc.) to send `Client-Id: ""`.
- Restored Twitch's real, public client IDs (the same ones used by the official web and Android clients, and by reference tools like DevilXD/TwitchDropsMiner).

## Fixes

- `Client-Id` header now carries a valid value on every direct API request, so Twitch's integrity check can pass without needing a rendered browser session.
- The browser (QWebEngine) fallback path is untouched and still available as a safety net, but should now rarely — if ever — be needed.

## Result

Version 2.2.30 should let Twitch Drop Farmer run fully streamless, with no browser window required for normal operation.
