# Twitch Drop Farmer v2.2.35

This release fixes a regression introduced in 2.2.34.

## Fixed

- **[REGRESSION from 2.2.34] Browser fallback stopped finding campaigns.** The on-disk persistent profile (`QWebEngineProfile("TwitchDropFarmer", app)`) added in 2.2.34 accumulates cookies/cache across *every* app restart. After dozens of restarts in one day of testing, the fallback started returning "no additional campaigns found" instead of the usual ~100+, plausibly because stale/conflicting Twitch session state confused its own page JavaScript. Reverted to an `isOffTheRecord=True` profile (confirmed with a real offscreen test), keeping only the low-risk part of 2.2.34: reusing the same profile instance within a single app run instead of creating a new one every poll cycle.

## Result

Version 2.2.35 restores the browser fallback's previous reliability while keeping the small efficiency win of not recreating the profile on every call.
