# Twitch Drop Farmer v2.2.37

This release turns the dashboard refresh button into an explicit, full-whitelist detail scan.

## Highlights

- The "Refresh dashboard" button previously just cleared image caches and ran a normal poll, still bound by the usual 8-campaigns-per-cycle detail-fetch budget. Selecting a different game right after refreshing would still show empty drop data most of the time.

## Added

- `fetch_campaigns(full_scan=True)` / `FarmEngine.poll(full_scan=True)`: an explicit request that lifts the per-cycle detail-fetch cap so every whitelisted campaign missing real per-drop data gets a detail-page visit in one go, not just a handful. The dashboard refresh button now calls this synchronously on the main thread (required for the browser fallback), disabling itself and logging start/finish since it can take a few minutes with a large whitelist. Verified with an isolated test: 19/20 campaigns get a detail visit in a full scan versus 8/20 in a normal cycle.

## Result

Version 2.2.37 lets the user proactively "warm up" the whole dashboard before switching between games, instead of only the actively farmed one ever showing real data.
