# Twitch Drop Farmer v2.2.34

- PT notes: [v2.2.34.pt-PT.md](v2.2.34.pt-PT.md)
- EN notes: [v2.2.34.en.md](v2.2.34.en.md)

Main points:

- The browser fallback now uses one persistent, shared profile instead of a fresh throwaway one per call.
- The Client-Integrity token Twitch's own JS computes inside the browser is now harvested and reused for direct HTTP requests for a few minutes.
- Wasted DropCampaignDetails retries are skipped when the integrity check is already confirmed blocked for the cycle.
- Confirmed the drops page has no server-rendered data to scrape without JS -- that option is a dead end.
