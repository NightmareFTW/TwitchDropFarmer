# Twitch Drop Farmer v2.2.36

This release prioritizes the browser fallback's limited detail-fetch budget toward the game the user actually cares about.

## Highlights

- With a 100+ campaign whitelist and only 8 campaigns getting a real detail-page visit per successful browser cycle, a freshly selected or auto-picked farm target could sit for many cycles with no real `next_drop_name`/`required_minutes` -- showing "Progresso da campanha: 0/0 min" and "Sem drops ativos" even while actively being farmed, because it simply hadn't had its turn in the soonest-ending queue yet.

## Added

- `TwitchClient.set_priority_game()`, called by the UI both when the user manually clicks a dashboard game (`_select_dashboard_game`) and whenever the engine auto-picks a farm target (`_refresh_farming_now`). `_select_browser_detail_candidate_ids` now puts that game's campaigns at the front of the detail-fetch queue, ahead of everything else regardless of end date. Verified with an isolated test: a campaign ending in 10 days now gets fetched before ones ending in hours, once marked as the priority.

## Result

Version 2.2.36 should make the selected/active farm target's drop panel populate meaningfully faster, though it still depends on a browser-fallback cycle actually succeeding (still gated by Twitch's integrity enforcement).
