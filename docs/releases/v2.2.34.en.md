# Twitch Drop Farmer v2.2.34

This release reduces how often the invisible browser fallback is actually needed, following up on the integrity-check work in earlier 2.2.x releases.

## Highlights

- Investigated four ways to reduce reliance on the QWebEngine fallback; implemented three, ruled out one after directly testing it.
- Ruled out: fetching `twitch.tv/drops/campaigns` as plain HTML. Confirmed by direct HTTP request -- the page has zero embedded campaign data; it's a fully client-rendered SPA with no server-side data injection to scrape.

## Added

- **Persistent browser profile**: `_get_browser_profile` now creates one named, on-disk `QWebEngineProfile` (verified `isOffTheRecord=False` against a live offscreen `QApplication`) shared across every browser-fallback call (listing, per-campaign detail, DOM scrape, drop claim), instead of a fresh anonymous profile every time. Cookies and local storage now persist across calls and app restarts, which should look like one continuous session to Twitch's anti-bot fingerprinting instead of a brand-new one every poll cycle.
- **Client-Integrity token harvesting and reuse**: the existing GQL-intercept script now also captures the `Client-Integrity` header Twitch's own JS attaches to its `/gql` requests inside the browser. That token is now offered on our own direct HTTP requests for up to ~4 minutes (cleared immediately if a request using it still gets rejected), so several poll cycles in a row can potentially succeed without invoking the browser at all.
- **DropCampaignDetails circuit breaker**: when `ViewerDropsDashboard` already confirmed the integrity check is blocked for this cycle and no usable token is available, the `DropCampaignDetails` retry loop (up to 2 identities x 2 header profiles x chunk -- up to ~8 wasted requests and 16 log lines per cycle) is skipped instead of repeating a failure we already know about.

## Result

Version 2.2.34 should reduce, but likely not eliminate, how often "a tentar fallback do browser no thread principal" fires -- the underlying cause remains Twitch's own intermittent integrity enforcement, which no client-side fix can fully control.
