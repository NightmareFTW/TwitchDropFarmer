# Twitch Drop Farmer v2.2.40

This release fixes the Filters tab showing an incomplete game list after a restart.

## Fixed

- The Filters tab (where you pick which games to whitelist) only showed the games already in the whitelist. That list is built from `fetch_campaigns`, and the campaign cache was memory-only — so after every app restart it started empty, and until a browser-fallback fetch succeeded you only saw the in-progress Inventory games, which are exactly the ones already being farmed/whitelisted.

## Added

- **Disk-persisted campaign cache** — `TwitchClient` now stores the campaign catalog (public metadata only: names, IDs, dates, progress; no credentials) in `~/.twitch-drop-farmer/campaign_cache.json`, written atomically and pruned of expired campaigns on load. The full game catalog now survives restarts, so the Filters tab immediately shows every game to pick a whitelist from, without waiting for a fresh browser fetch. The game list is also seeded from this cache at startup, before the first poll.

## Result

Version 2.2.40 makes the whitelist picker in Filters show the full game catalog straight after launch, instead of only the games already being farmed.
