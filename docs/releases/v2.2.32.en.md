# Twitch Drop Farmer v2.2.32

This release fixes a log-spam regression and reduces the latency impact of the per-campaign detail fetch introduced in 2.2.31.

## Highlights

- The DOM-scrape browser fallback polls via async `runJavaScript` calls. When the renderer is busy, multiple calls can be in flight at once; each one that later observes the same "give up" condition independently re-logged the stop message and re-triggered `finish()`.
- The 2.2.31 per-campaign detail-fetch phase (up to 40 page visits x 9s) was slow enough to delay the streamless heartbeat that actually accrues drop watch-time, visible as repeated "Heartbeat streamless adiado" and "Sem avanço após heartbeat" log lines.

## Fixed

- Added an idempotent `finished` guard to the DOM-scrape polling closure and to both browser GQL-intercept phases, so overlapping async callbacks no longer re-log or re-quit after the loop has already stopped. `finish()` now stops the poll timer immediately instead of waiting for the event loop to unwind.
- Reduced the detail-fetch phase to 8 campaigns per cycle (from 40) and 6s per-page timeout (from 9s), cutting its worst-case added latency from ~6 minutes to under a minute.

## Result

Version 2.2.32 keeps the invisible per-drop detail fetch from 2.2.31, but bounded so it can no longer meaningfully delay the heartbeat that drives actual farming progress, and without flooding the log.
