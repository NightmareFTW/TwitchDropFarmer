# Twitch Drop Farmer v2.2.31

This release extends the invisible browser fallback to fetch real per-drop campaign detail.

## Highlights

- Twitch's own integrity-check system has had prolonged, intermittent outages for direct GraphQL requests (independently confirmed by the DevilXD/TwitchDropsMiner maintainer, who hit the same failure in his own real browser). The invisible (headless, no window) browser fallback remains the reliable way to reach Twitch during those windows.
- That fallback previously only visited the campaign listing page, which returns summary data with no `timeBasedDrops` — leaving the active-drops panel and campaign progress empty for many active campaigns.

## Added

- `_campaigns_via_browser_gql_intercept` now also visits each active campaign's own detail URL (`/drops/campaigns?dropID=<id>`) on the same invisible page, capturing the real `DropCampaignDetails` response Twitch's own JS fires. Capped at 40 campaigns per cycle, soonest-ending campaigns first. No browser window is ever shown.

## Result

Version 2.2.31 should populate real drop progress even when Twitch's direct API integrity check is unavailable, at the cost of a slower refresh cycle when this path is used — an explicit tradeoff the user requested in favor of staying fully browser-window-free.
