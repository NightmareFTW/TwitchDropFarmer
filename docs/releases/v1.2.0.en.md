# Twitch Drop Farmer v1.2.0

This release improves campaign state accuracy, reorganizes filters for faster control, and makes farming behavior more predictable under real Twitch edge cases.

## Highlights

- New dashboard states for subscription-required campaigns, fully lost campaigns, and partially lost campaigns.
- Dedicated badges, ribbons, and tooltips to surface the real status of each game at a glance.
- Filters moved into sub-tabs with search and bulk actions (select/clear all and visible).
- Per-tab selected/total counters for better list control.
- Better farming panel continuity when a valid stream is temporarily unavailable.

## Technical Improvements

- Explicit detection for campaigns where all drops are already claimed.
- Explicit detection for campaigns that require an active subscription.
- Stream selection fallback when Twitch ACL payloads return labels that are not valid channel logins.
- Stricter ACL channel login token parsing.
- Improved browser/API campaign merge behavior to preserve progress/status metadata.

## Fixes

- Subscription-only campaigns are no longer treated as farmable watch-time targets.
- Fixed account-link fallback behavior when `self` connection payload is missing.
- Improved expired campaign representation to distinguish full loss vs partial loss.

## Result

The app now reflects real drops state more accurately, avoids false farming targets, and provides a faster, cleaner filter workflow.
